# RFP / spec docs

Five versions of the same project brief at decreasing levels of formality. Use these to shop the project to development studios and price-discover the real market for what we've built.

| File | When to use |
| --- | --- |
| `01-full-tech-spec.md` | Long, detailed RFP with explicit functional requirements, stack opinions, UX-per-surface notes. Hand to a studio that wants to scope precisely against a comprehensive spec. |
| `02-full-pilot-scope.md` | Same shape as #01 but scoped to **everything currently built** (post-PR #196). Use when asking "how much would it cost to rebuild what we already have, fully". |
| `03-product-brief-non-technical.md` | Short product-owner brief with no stack dictation. Use with studios you trust to bring engineering judgement. |
| `04-product-brief-with-tech-friend.md` | Middle version: product-owner voice plus light architecture pointers from a tech-friend consultation. The most balanced for most studios. |
| `05-pure-tech-spec.md` | Binding tech spec / SOW addendum. Hand to the engineering team after the contract is signed and discovery is done. |

All five describe the same product. The main axis of variation is **how much engineering judgement we delegate to the studio**.
